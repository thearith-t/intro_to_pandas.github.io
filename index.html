Part 1

Foundations of Pandas

Series · DataFrame · Loading Data

What is Pandas?

Pandas is the most popular Python library for working with tabular data — like Excel, but programmable.

  • Created in 2008 by Wes McKinney
  • Built on top of NumPy
  • Powers most data analysis in Python today

Why Pandas?

Without Pandas

rows = []
with open("data.csv") as f:
    for line in f:
        rows.append(line.split(","))

# manual filtering, totals,
# grouping, joining...

With Pandas

import pandas as pd
df = pd.read_csv("data.csv")
df.groupby("platform").mean()

One line replaces dozens. That’s the power of Pandas.

Pandas vs NumPy

import numpy as np
import pandas as pd

# NumPy — numbers only, no labels
arr = np.array([320, 1850, 4200])
print(arr)

# Pandas — labeled, mixed types
s = pd.Series([320, 1850, 4200],
              index=["Facebook", "Instagram", "TikTok"])
print(s)
[ 320 1850 4200]
Facebook      320
Instagram    1850
TikTok       4200
dtype: int64

Importing Pandas

The standard convention — always use pd:

import pandas as pd
import numpy as np

print("Pandas version:", pd.__version__)
Pandas version: 2.3.1

Convention

Every Python data project starts with import pandas as pd. Stick to the convention so others can read your code instantly.

Our Dataset

We will use store_social_media.csv — a small online store’s social media posts.

Each row is one post. Columns include:

  • date, platform, post_type, product
  • likes, comments, shares, views
  • ad_spend, revenue

Why This Dataset?

Real businesses live on social data — likes, reach, ad spend. Pandas is how analysts turn this into decisions.

Series

The 1-D Building Block

What is a Series?

A Series is a one-dimensional labeled array — like a single column of a spreadsheet.

import pandas as pd

likes = pd.Series([320, 1850, 4200, 540])
print(likes)
0     320
1    1850
2    4200
3     540
dtype: int64

Note

The left column (0, 1, 2, 3) is the index. The right column is the values.

Series with Custom Index

You can give the index meaningful labels:

likes = pd.Series(
    [320, 1850, 4200, 540],
    index=["Facebook", "Instagram", "TikTok", "YouTube"]
)
print(likes)
Facebook      320
Instagram    1850
TikTok       4200
YouTube       540
dtype: int64

Series from a Dictionary

A dict turns into a Series automatically — keys become the index:

post_likes = {
    "Facebook":  320,
    "Instagram": 1850,
    "TikTok":    4200,
    "YouTube":   2300
}
s = pd.Series(post_likes)
print(s)
Facebook      320
Instagram    1850
TikTok       4200
YouTube      2300
dtype: int64

Accessing Series Values

s = pd.Series(
    [320, 1850, 4200, 2300],
    index=["Facebook", "Instagram", "TikTok", "YouTube"]
)

print(s["TikTok"])      # by label
print(s[0])             # by position
print(s[["Facebook", "YouTube"]])  # multiple
4200
320
Facebook     320
YouTube     2300
dtype: int64

Series Operations

A Series behaves like a smart NumPy array:

likes = pd.Series([320, 1850, 4200, 2300],
    index=["Facebook", "Instagram", "TikTok", "YouTube"])

print("Sum:  ", likes.sum())
print("Mean: ", likes.mean())
print("Max:  ", likes.max())
print("Min:  ", likes.min())
Sum:   8670
Mean:  2167.5
Max:   4200
Min:   320

DataFrame

The 2-D Powerhouse

What is a DataFrame?

A DataFrame is a 2-D labeled table — rows and columns — like an Excel sheet or SQL table.

Think of it as many Series glued together, sharing the same index.

df = pd.DataFrame({
    "platform": ["Facebook", "Instagram", "TikTok"],
    "likes":    [320, 1850, 4200],
    "shares":   [18, 340, 910]
})
print(df)
    platform  likes  shares
0   Facebook    320      18
1  Instagram   1850     340
2     TikTok   4200     910

DataFrame from a Dictionary

The keys are column names, the lists are column values:

data = {
    "post_id":  [1, 2, 3, 4],
    "platform": ["Facebook", "Instagram", "TikTok", "YouTube"],
    "likes":    [320, 1850, 4200, 2300],
    "revenue":  [210, 650, 820, 540]
}
df = pd.DataFrame(data)
print(df)

1        2  Instagram   1850      650
2        3     TikTok   4200      820
3        4    YouTube   2300      540

Missing Data

The Real World is Messy

What is Missing Data?

In Pandas, missing values are shown as NaN (Not a Number).

They appear when:

  • A cell was empty in the CSV
  • A calculation was undefined (e.g. 0/0)
  • Data was lost or never collected

Our store dataset has a few real gaps — let’s find them.

fillna() Strategies

Strategy When to use
fillna(0) Counts where missing means “none”
fillna(mean) Continuous numbers (price, spend)
fillna(median) When outliers skew the mean
fillna("Unknown") Text / category columns

What is groupby()?

groupby() splits the table into chunks based on a column, then runs a calculation on each chunk.

Split → Apply → Combine

  1. Split rows by a category
  2. Apply a function (sum, mean…) to each group
  3. Combine results into a new table

This is how analysts turn raw data into insight.

Real Business Question

Which platform gives the best return on ad spend?

roi = df.groupby("platform").agg(
    total_revenue  = ("revenue", "sum"),
    total_spend    = ("ad_spend", "sum")
)
roi["roi"] = (roi["total_revenue"] / roi["total_spend"]).round(2)
roi.sort_values("roi", ascending=False)
total_revenue total_spend roi
platform

Instagram 5890.0 354.0 16.64
